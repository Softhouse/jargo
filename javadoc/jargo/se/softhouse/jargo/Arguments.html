Class Arguments

java.lang.Object
  se.softhouse.jargo.Arguments

public final class Arguments
extends Object

 Used as a starting point to create Argument instances.
 
 The produced arguments can be passed to
 CommandLineParser.withArguments(Argument...) to group several
 Arguments together. If only one argument should be
 parsed, ArgumentBuilder.parse(String...) can be used instead.

helpArgument

public static Argument<?> helpArgument(String mandatoryName,
                                       String... optionalNames)
 Makes the CommandLineParser it's passed to throw a helpful ArgumentException
 with usage explaining either the full program or a specific Argument/Command.
 The resulting Usage is currently only accessible through the ArgumentException.getMessageAndUsage() method.
 This suits most use-cases as there's usually a catch block for ArgumentException anyway where the Usage is printed
 
 Supported usages:
 
    
 
  • "program -h" - Gives Usage for the whole program
    • 
 
  • "program -h --number" Gives Usage for the "--number"Argument
    • 
 
  • "program commit -h" Gives Usage for the commit Command
    • 
 
  • "program -h commit" Also gives Usage for the commit Command
    • 
 
  • "program commit -h --amend" Gives Usage for the --amend Argument for the commit Command
    • 
 
 

 
 If given an unknown argument name, an ArgumentException is thrown.





Parameters:
mandatoryName - "-h" in the above example
optionalNames - for example "--help"

Returns:
an Argument that can be passed to CommandLineParser.withArguments(Argument...)

@CheckReturnValue

public static ArgumentBuilder.OptionArgumentBuilder optionArgument(String mandatoryName,
                                                                                            String... optionalNames)
 Creates an optional (flag) Argument where the sole existence of the name on the command
 line matters. This is the only place to have an ArgumentBuilder.arity(int) of zero.
 
 The default value is printed with Describers.booleanAsEnabledDisabled().
 This can be changed with ArgumentBuilder.defaultValueDescriber(Describer).





Parameters:
mandatoryName - the first name that enables this option
optionalNames - aliases that also enables this option
See Also:
