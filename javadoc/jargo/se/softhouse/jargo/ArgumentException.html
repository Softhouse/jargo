@NotThreadSafe
public abstract class ArgumentException
extends RuntimeException

Indicates that something went wrong in a parsing. Typical causes include:

  • Missing parameters
  • Unknown arguments, if it's close enough to a known argument the error message will contain suggestions
  • Missing required arguments
  • Invalid arguments, thrown from parse
  • Repetition of argument that hasn't specified repeated
The typical remedy action is to present getMessageAndUsage() to the user so he is informed about what he did wrong.
As all Exceptions should be, ArgumentException is Serializable so usage is available after deserialization.
