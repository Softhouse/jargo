closestMatch

@Nonnull
@CheckReturnValue
public static String closestMatch(String input,
                                                           Iterable<String> validOptions)
 Returns the String in validOptions that input has the shortest
 levenshtein distance to.
 
 For example when given "stats" as input and "status", "help",
 "action" as validOptions, "status" is returned.
 
 Current performance characteristics:
 n = length of input
 m = average string length of the strings in validOptions
 s = amount of validOptions
 
 complexity = n * m * s = O(n3)
 
 So try to limit the number of valid options...






Throws:

IllegalArgumentException - if validOptions is empty

closestMatches

@Nonnull
@CheckReturnValue
public static List<String> closestMatches(String input,
                                                                   Iterable<String> validOptions,
                                                                   int maximumDistance)
 Returns a sorted List where the first entry is the String in validOptions that's closest in terms of
 levenshtein distance to input, or an empty list if no options within distance can be found.
 
 For example when given "stats" as input and "status", "staging",
 "stage" as validOptions, and 4 as maximumDistance, "status", "stage", "staging" is returned.
 
 Only values with a distance less than or equal to maximumDistance will be included in the result.
 
 The returned list is modifiable.
